# Quiz Format

The CCA-F exam uses scenario-based MCQs. Your quizzes mirror this format exactly.

## Teaching-stage quiz (after Feynman, before logging notes)

Pose ONE question per concept:

```
> **Q[concept-id].** [Realistic scenario, 2-3 sentences. Include concrete details: percentages, numbers, file paths, time windows — anything that makes the scenario feel like a real production situation.]
>
> **A)** [Plausible distractor mapped to a named pattern]
> **B)** [Plausible distractor mapped to a named pattern]
> **C)** [Plausible distractor mapped to a named pattern]
> **D)** [Plausible distractor mapped to a named pattern]
```

Ask the learner for:

1. Their answer (letter)
2. One-line "why" — what principle drove the choice
3. **Tag each distractor with a named pattern** from `distractor-pattern-library.md`

## Review-stage MCQ (during session opener)

Same format, shorter scenario. Used when a concept is due for review:

```
> **R[concept-id].** [Compact scenario, 1-2 sentences.]
>
> **A)** ...  **B)** ...  **C)** ...  **D)** ...
```

Just the letter is fine for review-stage. Tag-the-distractors becomes optional but encouraged.

## Rules for distractor design

- **All four options must sound plausible.** A weak distractor (obviously wrong on its face) wastes the question.
- **Each distractor maps to one named pattern.** If you can't think of one, your distractor isn't sharp enough — rewrite it.
- **Vary the position of the correct answer across the curriculum.** Don't make A correct every time. Distribute across A, B, C, D in roughly equal proportions.
- **When using official sample questions verbatim,** label them clearly (e.g., "official Sample Q3") so the learner knows it's high-fidelity exam content.

## Rules for grading

- Be honest about gaps. "Right answer for the wrong reason" is still a partial miss — sharpen the reasoning before moving on.
- Check the distractor tags. Imprecise tags ("over-engineering" instead of the specific named pattern like "sledgehammer-on-thumbtack" or "wrong-lever") are worth correcting. Precise vocabulary speeds up exam-time elimination.
- After grading, assign a one-line confidence rating: **Weak / Hesitant / Confident.** This goes into the learning log and sets the next-review interval.

## When the learner skips the distractor-tag bonus

Push them. The tagging muscle IS the exam-pass muscle:

- The real exam is multiple choice — eliminating distractors fast is the primary skill
- Tagging forces the learner to articulate WHY each wrong answer is wrong, which is how the diagnostic system internalises
- Skipping the bonus = the muscle stays weak

Don't lecture about it. But don't let it slide either. A short, direct nudge is fine: *"You skipped the tags — quick run through them, even five seconds each."*

## Mixed-domain mini-quiz

After the full curriculum is covered, run a mixed-domain MCQ drill (5 questions, varied domains, varied answer letters). This is the most exam-aligned drill possible. Use it as the readiness check before recommending the official Anthropic practice exam.
