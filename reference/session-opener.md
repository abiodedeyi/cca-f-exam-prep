# Session Opener

Every session opens the same way. Follow these steps before any new teaching.

## Step 1: Read the learning log

Open the learner's `learning-log.md`. Identify all rows where `Next Due ≤ today`.

## Step 2: Review what's due (≈5 min total)

For each due concept:

- Pose 1–2 targeted MCQs (review-style — see `quiz-format.md`)
- Wait for the learner's answer
- Grade honestly

**Important:** review MCQs target the SPECIFIC items the learner missed earlier, or the highest-value testable mechanic for that concept. They are NOT broad re-teaching. If the learner is solid, this should take 30 seconds per concept.

## Step 3: Drill watch-list items if their parent concept is due

The watch list inside `learning-log.md` lists specific gaps (e.g., "text-content-as-completion anti-pattern" tied to concept 1.1). When that gap's parent concept comes due today, drill the gap specifically alongside the regular review.

## Step 4: Update the log based on review performance

For each concept reviewed:

- **Cleared cleanly** → double the next interval
- **Missed something specific** → drop to Hesitant (3 days) AND add to watch list if the gap is precise
- **Completely blanked** → Weak (1 day)

## Step 5: Move to new material

Only after reviews are done. Find the next unchecked task statement and start the 4-step teaching loop (`study-method.md`).

## Edge case: nothing due today

Tell the learner so, then proceed directly to teaching the next concept.

## Edge case: practice exam has been taken

When the learner has taken the official Anthropic practice exam:

- Map their missed topics to specific concepts in the curriculum
- Bump the spaced-rep next-due date for those specific concepts to ASAP (so they're re-drilled at the next session)
- Add the specific weak items to the watch list
- Don't override the schedule globally — just bump the items the practice exam surfaced

## Edge case: cross-concept interference detected

If a learner misses a review specifically because they crossed wires with a different concept (e.g., confusing parallel spawning with `context: fork`), this is a HIGH-VALUE catch. Add a watch-list item naming the *specific confusion* (not just the concept) so the next review can re-test the precise distinction.
